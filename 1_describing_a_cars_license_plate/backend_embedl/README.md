Backend FastAPI for analyzing vehicle license plates using the **`embedl/Cosmos-Reason2-2B-W4A16`** (Vision Language Model optimized for `Edge AI`).

Accepts image or video uploads and returns a detailed description (country, number, format, brand, etc.) generated by the VLM. The processing is done in the background, with polling by the frontend.

---

## <font color="red">Structure</font>

```
backend_embedl/
|-- app/
|   |-- main.py                # FastAPI, CORS, lifespan events
|   |-- config.py              # Settings via pydantic-settings

|   |   |-- analysis.py        # POST /api/v1/jobs, GET /api/v1/jobs/{id}
|   |-- core/
|   |   |-- model.py           # EmbedlVLM (manages the VLM in memory)
|   |   |-- analyzer.py        # Inference pipeline for image/video
|   |   |-- prompts.py         # System prompts in English
|   |-- schemas/
|   |   |-- analysis.py        # Pydantic models (request/response schemas)
|   |-- services/
|   |   |-- job_manager.py     # Job store thread-safe in-memory
|   |   |-- file_handler.py    # Saves uploads to data/uploads
|   |-- utils/
|       |-- memory.py          # cleanup_memory, resolve_device/dtype
|-- scripts/
|   |-- download_images_and_videos.py
|-- data/
|   |-- images/                # Images downloaded for testing
|   |-- videos/                # Videos downloaded for testing
|   |-- uploads/               # Temporary uploads via API
|-- .env.example
|-- run.sh
|-- README.md
```

## <font color="red">Endpoints</font>

| Method | Route                             | Description                                   |
|--------|-----------------------------------|-----------------------------------------------|
| GET    | `/`                               | Basic information about the API                |
| GET    | `/health`                         | Status of the service and the model            |

## <font color="red">How to Run?</font>

### <font color="blue">`1.` Install dependencies (at the root of the repository)</font>
```bash
uv sync
```

### <font color="blue">`2.` Configure environment variables</font>
```bash
cd 1_describing_a_cars_license_plate/backend_embedl
cp .env.example .env
# edit .env as needed
```

### <font color="blue">`3.` Start the server</font>
```bash
./run.sh
```

Or manually:
```bash
uv run uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

Access:
- API: http://localhost:8000
- Documentation: http://localhost:8000/docs

## <font color="red">Usage Flow (frontend React)</font>

```
1. POST /api/v1/jobs (multipart/form-data with 'file')
   -> immediate response: { job_id, status: "queued", poll_url, ... }

2. Polling in loop every 2 seconds:
   GET /api/v1/jobs/{job_id}
   -> { status: "processing" }
   -> { status: "processing" }
   -> { status: "completed", result: { description, ... } }

3. (Optional) DELETE /api/v1/jobs/{job_id} to clear
```

### <font color="blue">Example with curl</font>

**Upload image:**
```bash
curl -X POST http://localhost:8000/api/v1/jobs \
  -F "file=@path/to/your/image.jpg"
```
After the job is created, you can query the status with the job_id:

**Query status:**
```bash
curl http://localhost:8000/api/v1/jobs/<job_id>
```

## <font color="red">Important Configurations (`.env`)</font>

| Variable             | Default                             | Description                            |
|----------------------|-------------------------------------|----------------------------------------|
| `DEVICE`             | `auto`                              | `auto` / `cuda` / `cpu` / `mps`        |
| `DTYPE`              | `auto`                              | `float16` on GPU, `float32` on CPU     |
| `MODEL_NAME`         | `embedl/Cosmos-Reason2-2B-W4A16`    | VLM model to load                       |
| `MAX_UPLOAD_MB`      | `100`                               | Upload size limit                       |
| `MAX_NEW_TOKENS`     | `300`                               | Maximum tokens in generated response    |
| `JOB_TTL_MINUTES`    | `30`                                | Time before cleaning finished jobs       |
| `CORS_ORIGINS`       | `localhost:3000, 5173, 4173`        | Allowed origins by CORS                 |

## <font color="red">Edge AI</font>

This backend was designed for deployment on edge devices (Jetson, Raspberry Pi 5, etc.):

- Quantized model **W4A16** (4-bit weights, 16-bit activations) -> ~75% less memory
- Singleton VLM (loads once, avoids reload per request)
- Job store **in-memory** (no Redis/Celery)
- `BackgroundTasks` native to FastAPI (no external queue)
- Aggressive memory cleanup after each inference
- Device and dtype configurable via `.env`
- Minimal dependencies (FastAPI + Uvicorn + Torch + Transformers)
